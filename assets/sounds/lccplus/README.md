# LCC+ bundled sound assets

These WAV files are project-owned default sounds for the LCC+ `sound` trap slots:

| Slot | Alias | File |
|---:|---|---|
| 0 | `ding` | `ding.wav` |
| 1 | `deep` | `deep.wav` |
| 2 | `bop` | `bop.wav` |
| 3 | `doink` | `doink.wav` |
| 4 | `beep` | `beep.wav` |

## Provenance

The files are deterministic PCM WAV tones generated by
`scripts/generate-lccplus-sounds.js`. They are not copied from an operating
system sound theme or third-party sound library.

They are intended to be distributed with lccjs under the same project terms as
the surrounding repository files. Regenerate them with:

```bash
node scripts/generate-lccplus-sounds.js
```

## Runtime fallback order

By default, the LCC+ runtime uses the bundled WAV assets in this directory so
the fallback sounds are stable across machines.

Set `SOUND_FILES_FROM_SYSTEM=1` in `.env` or the process environment to prefer
local filesystem sounds. In that mode, each slot resolves in this order:

1. user-provided environment variable path (`LCCPLUS_SOUND_*`);
2. known OS desktop sound path, if present;
3. bundled project WAV asset from this directory;
4. terminal BEL (`\x07`) if no file/player works.
