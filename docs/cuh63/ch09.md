## ex0901.a — Static struct: field access and pointer access

A `struct Point { int x; int y; }` is allocated as two consecutive `.word` / `.zero` slots at a label. Fields are accessed by label-plus-offset for direct access, or by pointer dereference with an offset for arrow access.

C equivalent:
```c
struct Point { int x; int y; };
struct Point a;
struct Point *p;
int main() {
    a.x = 1;    // st r0, a     (offset 0)
    a.y = 2;    // st r0, a+1   (offset 1)
    p = &a;     // lea r0, a; st r0, p
    printf("%d\n", a.y);    // ld r0, a+1
    printf("%d\n", p->y);   // ld r0, p; ldr r0, r0, 1
    printf("%d\n", (*p).y); // same as p->y
}
```

Field access patterns:
| Access | Assembly |
|--------|---------|
| `a.x` (direct) | `ld r0, a` / `st r0, a` |
| `a.y` (direct) | `ld r0, a+1` / `st r0, a+1` |
| `p->y` (pointer) | `ld r0, p; ldr r0, r0, 1` |

## ex0902.a — Dynamic struct allocation

Uses an inline `malloc` (bump allocator) to allocate a struct at runtime. `malloc` receives the size in `r1` and returns the start address in `r0`.

C equivalent:
```c
struct Point *p = malloc(sizeof(struct Point));
p->y = 5;
printf("%d\n", p->y);
```

Assembly calling convention for this malloc:
```asm
mov r1, 2    ; sizeof(struct Point) = 2 words
bl malloc
st r0, p     ; p = returned address
```

The inline malloc implementation:
```asm
malloc:  ld r0, @avail      ; start of free area
         add r1, r0, r1     ; bump by size
         st r1, @avail      ; update free pointer
         ret                ; return old start in r0
@avail:  .word *+1          ; initially points to the word after itself
```

## ex0903.a — Passing structs: by value vs. by pointer

Compares two calling conventions for struct arguments:

**Pass by value (`f`):** all struct fields are pushed individually onto the stack. The callee receives its own copy.
```asm
ld r0, a+1   ; push a.y first (rightmost)
push r0
ld r0, a     ; push a.x
push r0
bl f
add sp, sp, 2
```

**Pass by pointer (`g`):** a single address is pushed. The callee uses `ldr r1, fp, 2` then `ldr r0, r1, 0` / `ldr r0, r1, 1`.
```asm
lea r0, a    ; push &a
push r0
bl g
add sp, sp, 1
```

Key contrast: pass-by-value copies grow linearly with struct size; pass-by-pointer is always one word.
