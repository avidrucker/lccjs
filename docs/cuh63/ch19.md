# Chapter 19 — Address Semantics

Source files from the cuh63 package. Chapter 19 examines how addresses are created, stored, propagated, and destroyed (overwritten) at the machine level.

> **ISA reference:** [docs/lcc-isa.md](../lcc-isa.md)

## ex1901.a — Creation, propagation, and destruction of addresses

A minimal program that demonstrates the three things that can happen to a machine-word address:

**Creation:** `.word d` assembles `a` to the 16-bit address of `d` (not the value stored at `d`).
```asm
a: .word d    ; a holds the address of d (0x0008)
```

**Propagation:** loading an address into a register and storing it elsewhere.
```asm
ld r0, a     ; r0 = address of d (0x0008)
st r0, b     ; b now also holds that address
```

**Destruction:** overwriting an address slot with a non-address (plain integer).
```asm
ld r1, c     ; r1 = 5  (a plain integer, not an address)
st r1, a     ; a now holds 5 — the address it used to hold is gone
```

Full program trace:

| Statement | Effect |
|-----------|--------|
| `ld r0, a` | r0 = 0x0008 (address of d) |
| `st r0, b` | mem[b] = 0x0008 — address propagated to b |
| `ld r1, c` | r1 = 5 — a plain constant |
| `st r1, a` | mem[a] = 5 — address in a overwritten; a no longer points to d |
| `halt` | stop |

Memory layout:
| label | address | initial value | after program |
|-------|---------|---------------|---------------|
| a | 0x0004 | 0x0008 (addr of d) | 5 (destroyed) |
| b | 0x0005 | 3 | 0x0008 (propagated) |
| c | 0x0006 | 5 | 5 |
| d | 0x0008 | 17 | 17 (unchanged) |

Key insight: addresses are just 16-bit words — there is no runtime tag distinguishing a pointer from an integer. Once overwritten, an address is gone.
