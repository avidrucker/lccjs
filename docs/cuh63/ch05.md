## ex0501.a — Global variables

Global variables are allocated as `.word` labels at file scope. Access uses `ld`/`st` by label name.

C equivalent:
```c
int x = 2, y;
int main() { y = x; printf("%d\n", y); return 0; }
```

Assembly access pattern:
```asm
ld r0, x     ; load global
st r0, y     ; store global
```

Key point: globals persist for the lifetime of the program and are accessible from any function by label.

## ex0502.a — Dynamic local variables

Local variables live on the stack, allocated by adjusting `sp` downward. Access is via `fp`-relative addressing with negative offsets.

C equivalent:
```c
void f(int a) {
    int x = 1, y;
    y = x + a;
    printf("%d\n", y);
}
```

Stack layout in `f` (after prologue and local allocation):
| fp offset | contents |
|-----------|---------|
| `fp+2` | argument `a` |
| `fp+1` | saved `fp` |
| `fp+0` | saved `lr` |
| `fp-1` | local `x` (initialized to 1 with `push r0`) |
| `fp-2` | local `y` (allocated with `sub sp, sp, 1`) |

Access pattern:
```asm
ldr r0, fp, -1   ; read x
str r0, fp, -2   ; write y
```

## ex0503.a — Static local variables

Static locals are allocated as named `.word` labels at file scope (not on the stack). The assembler uses mangled names (`@s0_x`, `@s1_x`) to distinguish multiple statics with the same source name.

C equivalent:
```c
int x = 5;            // global x
void f() { static int x; printf("%d\n", x); }
int main() { static int x = 3; printf("%d\n", x); f(); g(); return 0; }
```

Assembly representation:
```asm
@s0_x:  .word 0    ; static x inside f (zero-initialized)
@s1_x:  .word 3    ; static x inside main (initialized to 3)
```

Access uses `ld`/`st` by the mangled static label — identical to global access, just with a compiler-generated name.

Key contrast:
- **global** `x` — accessed as `ld r0, x`
- **static local** `x` in `f` — accessed as `ld r0, @s0_x`
- **static local** `x` in `main` — accessed as `ld r0, @s1_x`
- Also demonstrates `jsr` as a synonym for `bl`.
