## ex0401.a — Calling a function with arguments

Demonstrates the full LCC calling convention for passing two integer arguments.

C equivalent:
```c
void f(int x, int y) { printf("%d\n", x + y); }
int main() { f(1, 2); return 0; }
```

Calling sequence (push args right-to-left):
```asm
mov r0, 2    ; push y first (rightmost arg)
push r0
mov r0, 1    ; push x second
push r0
bl f
add sp, sp, 2  ; pop args
```

Stack frame layout in `f` (relative to `fp`):
| fp offset | contents |
|-----------|---------|
| `fp+3` | y (second arg, pushed first) |
| `fp+2` | x (first arg, pushed last) |
| `fp+1` | saved fp |
| `fp+0` | saved lr |

Callee prologue/epilogue pattern:
```asm
push lr / push fp / mov fp, sp
...
mov sp, fp / pop fp / pop lr / ret
```

Key instructions: `push`, `pop`, `bl`, `ret`, `mov`, `ldr`, `str`, `add`, `dout`, `nl`

## ex0402.a — Returning a value

Shows how a function returns a value in `r0`.

C equivalent:
```c
int f() { return 5; }
int main() { printf("%d\n", f()); return 0; }
```

The callee places the return value in `r0` before restoring the frame and returning. The caller uses `r0` directly after `bl f`.

Key point: no argument cleanup needed when there are no arguments.
