## ex1201.a — `argc` and `argv`

Demonstrates how the OS (via the `startup` stub) passes command-line arguments to `main`. Unlike the other examples, this file uses `.global main` (no `startup` wrapper) so the linker can position `main` at the entry point and the OS can call it with standard C `argc`/`argv`.

C equivalent:
```c
int main(int argc, char *argv[]) {
    int i = argc - 1;
    while (i >= 0) { printf("%s\n", argv[i]); i--; }
    return 0;
}
```

Stack frame layout for `main` (passed by the OS/startup):
| fp offset | contents |
|-----------|---------|
| `fp+3` | `argv` (pointer to array of string pointers) |
| `fp+2` | `argc` (argument count) |
| `fp+1` | saved `fp` |
| `fp+0` | saved `lr` |
| `fp-1` | local `i` |

Accessing `argv[i]`:
```asm
ldr r0, fp, 3    ; r0 = argv (base of pointer array)
ldr r1, fp, -1   ; r1 = i
add r0, r0, r1   ; r0 = &argv[i]
ldr r0, r0, 0    ; r0 = argv[i]  (the string pointer)
sout r0
```

The example prints arguments in reverse order (from `argc-1` down to 0).
