# Chapter 14 — C++ References

Source files from the cuh63 package. Chapter 14 covers C++ reference parameters and reference variables, showing how the compiler implements them at the machine level (as hidden pointers).

> **ISA reference:** [docs/lcc-isa.md](../lcc-isa.md)

---

## ex1401.a — Reference parameters (`int &a`)

A C++ reference parameter compiles identically to a C pointer parameter. The compiler automatically takes the address at the call site and automatically dereferences in the callee.

C++ equivalent:
```cpp
int x = 5;
void f(int &a) { a = a + 1; }
int main() {
    cout << x;  // 5
    f(x);
    cout << x;  // 6
}
```

Calling sequence (compiler inserts `lea`):
```asm
lea r0, x    ; push &x  (address, not value)
push r0
bl @f$ri     ; mangled name for f(int&)
add sp, sp, 1
```

Callee dereferences on every access:
```asm
ldr r0, fp, 2    ; r0 = pointer to a
ldr r0, r0, 0    ; r0 = *ptr  (current value of a)
add r0, r0, 1
ldr r1, fp, 2
str r0, r1, 0    ; *ptr = r0
```

Name mangling: `@f$ri` — `f` with a reference-to-int parameter.

## ex1402.a — Reference variables (global and local)

A C++ reference variable is stored as a pointer. Initializing `int &xr = x` stores `&x` in `xr`; every subsequent use of `xr` dereferences that pointer.

C++ equivalent:
```cpp
int x = 5;
int &xr = x;         // global reference
int main() {
    int y = 7;
    int &yr = y;     // local reference (stack)
    cout << x;       // direct
    cout << xr;      // dereference xr → x
    cout << y;       // direct
    cout << yr;      // dereference yr → y
}
```

Global reference at file scope:
```asm
xr: .word x      ; xr holds the address of x
```
Access: `ld r0, xr; ldr r0, r0, 0` (load pointer, then dereference).

Local reference on stack:
```asm
add r0, fp, -1   ; address of y
push r0          ; yr = &y
```
Access: `ldr r0, fp, -2; ldr r0, r0, 0`.

## ex1403.a — Reference variables (near-identical variant)

Nearly identical to ex1402. The only visible difference is `dout r0` vs. `dout` (implicit `r0`) in one instruction — illustrating that omitting the register in a trap defaults to `r0`.

Used to confirm that the compiler generates consistent code for the same C++ construct.
