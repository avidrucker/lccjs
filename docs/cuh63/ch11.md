## ex1101.a — Multiplication: slow vs. optimized

**`slowmul(x, y)`** — repeated addition: adds `x` to a running product `y` times. O(y) steps.

```asm
mov r0, 0          ; product = 0
@L0: and r2, r2, r2  ; while (y != 0)
     brz @L1
     add r0, r0, r1  ;   product += x
     sub r2, r2, 1   ;   y--
     br @L0
@L1: ret
```

**`mul(x, y)`** — binary method (Russian peasant multiplication): examines each bit of `y`, conditionally adds `x` to the product, then shifts `y` right and `x` left. O(log y) steps.

```asm
@L2: and r2, r2, r2    ; while (y != 0)
     brz @L3
     and r3, r2, 1     ; if (y & 1)
     brz @L4
     add r0, r0, r1    ;   product += x
@L4: srl r2, 1         ; y >>= 1
     sll r1, 1         ; x <<= 1
     br @L2
@L3: ret
```

Key instructions: `and`, `srl` (shift right logical), `sll` (shift left logical)

Both functions use the register-argument convention (no stack frame): `r1 = x`, `r2 = y`, result in `r0`.

## ex1102.a — Division: optimized by repeated subtraction

**`div(x, y)`** — subtracts the divisor from the dividend repeatedly, counting iterations until the result goes negative.

```asm
mov r0, 0           ; quotient = 0
@L0:
    sub r1, r1, r2  ; x -= y
    brn @L1         ; if x < 0, done
    add r0, r0, 1   ; quotient++
    br @L0
@L1: ret            ; return quotient
```

No hardware division instruction is used. The LCC ISA does have a `div` machine instruction, but this example implements division from first principles to illustrate the algorithm.

Also uses the register-argument convention: `r1 = x` (dividend), `r2 = y` (divisor), `r0 = quotient`.
