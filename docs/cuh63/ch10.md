## ex1001.a — Accessing arrays (global and local, constant and variable index)

Shows all four combinations of (global vs. local) × (constant vs. variable index).

C equivalent:
```c
int ga[10], x = 3;
int main() {
    int la[10];
    ga[2] = 10;    // constant-index global
    ga[x] = 11;    // variable-index global
    la[2] = 12;    // constant-index local
    la[x] = 13;    // variable-index local
}
```

**Constant-index global** — assembler computes address at assembly time:
```asm
mov r0, 10
st r0, ga+2        ; address is ga+2, resolved at assemble time
```

**Variable-index global** — base address + runtime index:
```asm
lea r1, ga         ; r1 = &ga[0]
ld  r2, x          ; r2 = index
add r1, r1, r2     ; r1 = &ga[x]
str r0, r1, 0      ; ga[x] = r0
```

**Constant-index local** — fp-relative with compile-time offset:  
(`la` starts at `fp-10` for a 10-word array: `la[2]` is at `fp-8`)
```asm
str r0, fp, -8
```

**Variable-index local** — base address + runtime index:
```asm
add r1, fp, -10    ; r1 = &la[0]
ld  r2, x
add r1, r1, r2     ; r1 = &la[x]
str r0, r1, 0
```

## ex1002.a — Passing arrays (four equivalent forms)

Demonstrates that `int z[]` and `int *z` are identical as parameter types, and that `z[1]` and `*(z+1)` produce identical code.

Four functions that all do the same thing (print `a[1]`):

| Function | Declaration | Access |
|----------|-------------|--------|
| `f1` | `void f1(int z[])` | `z[1]` |
| `f2` | `void f2(int *z)` | `*(z+1)` |
| `f3` | `void f3(int z[])` | `*(z+1)` |
| `f4` | `void f4(int *z)` | `z[1]` |

All four compile to:
```asm
ldr r0, fp, 2   ; r0 = z (the pointer/base address)
ldr r0, r0, 1   ; r0 = *(z+1)  (ldr with offset 1)
```

Arrays decay to pointers when passed to functions — there is no distinction at the machine level.

## ex1003.a — Strings: all allocation forms + `mystrcpy`

Shows all five ways to declare/create a string in C and their assembly equivalents:

| C declaration | Assembly |
|--------------|---------|
| `char g[] = "AX"` | `g: .string "AX"` (global array) |
| `char *p = "BX"` | `p: .word @m0` / `@m0: .string "BX"` |
| `char c[] = "CX"` | push chars individually onto stack |
| `char *q = "DX"` | `lea r0, @m1; push r0` (pointer to literal) |
| `char *r;  r = "EX"` | `lea r0, @m2; str r0, fp, -5` |

**`mystrcpy(char *p, char *q)`** — character-by-character copy loop:
```asm
@L0:
    ldr r0, fp, 3    ; load *q
    ldr r0, r0, 0
    ldr r1, fp, 2    ; *p = *q
    str r0, r1, 0
    cmp r0, 0        ; if (*q == 0) break
    brz @L1
    ; increment both pointers
    ldr r0, fp, 2 / add r0, r0, 1 / str r0, fp, 2
    ldr r0, fp, 3 / add r0, r0, 1 / str r0, fp, 3
    br @L0
@L1:
```

The null terminator (`0`) is copied as part of the loop body before the break — ensuring the destination is properly terminated.
