## ex0701.a — Pointer to a global variable

Demonstrates taking the address of a global with `lea`, storing it in a pointer, dereferencing it for a read, and writing through it.

C equivalent:
```c
int *p, x = 7;
int main() {
    p = &x;              // lea r0, x; st r0, p
    printf("%d\n", *p);  // ld r0, p; ldr r0, r0, 0; dout r0
    *p = 8;              // mov r0, 8; ld r1, p; str r0, r1, 0
    printf("%d\n", x);   // ld r0, x; dout r0
}
```

Dereference pattern (load through pointer):
```asm
ld r0, p       ; r0 = address stored in p
ldr r0, r0, 0  ; r0 = mem[r0 + 0]  (i.e., *p)
```

Write-through pattern:
```asm
mov r0, 8      ; value to store
ld r1, p       ; r1 = address in p
str r0, r1, 0  ; mem[r1] = r0
```

## ex0702.a — Pointer to a local variable

Gets the address of a stack-allocated local using `add r0, fp, -2` (adds the fp-relative offset to fp to produce an address).

C equivalent:
```c
int main() {
    int *p, x = 7;
    p = &x;              // add r0, fp, -2; str r0, fp, -1
    printf("%d\n", *p);  // ldr r0, fp, -1; ldr r0, r0, 0
    *p = 8;
    printf("%d\n", x);
}
```

Key pattern — get address of local variable:
```asm
add r0, fp, -2   ; r0 = &x  (fp-relative address of x)
str r0, fp, -1   ; store into p
```

## ex0703.a — Pointer to a function

Stores the address of a function (`lea r0, f; st r0, p`) and calls it indirectly with `blr r0`.

C equivalent:
```c
int (*p)(int, int);
int f(int x, int y) { return x+y; }
int main() {
    int sum = f(1, 2);
    p = f;
    sum = p(1, 2);   // indirect call
}
```

Indirect call pattern:
```asm
ld r0, p     ; load function pointer into r0
blr r0       ; call: lr = pc; pc = r0
```

`blr` (branch-and-link-to-register) is the indirect counterpart of `bl`.
