# Chapter 8 — Parameter Passing Mechanisms

Source files from the cuh63 package. This chapter covers all four classical parameter passing mechanisms, using LCC assembly to expose exactly what each one does at the machine level.

> **ISA reference:** [docs/lcc-isa.md](../lcc-isa.md)

> **Note:** ex0805.a is absent from the cuh63 package.

---

## ex0801.a — Pass by value

A copy of the argument is pushed onto the stack. The callee modifies its local copy; the caller's original is unchanged.

C equivalent:
```c
int x = 7;
void f(int a) { a = a + 1; }   // modifies copy only
int main() { f(x); printf("%d\n", x); }  // prints 7
```

The callee's `str r0, fp, 2` writes only to the stack slot, not back to `x`.

---

## ex0802.a — Pass by address (C pointer semantics)

The *address* of the variable is passed. The callee dereferences the pointer to modify the caller's original.

C equivalent:
```c
int x = 7;
void f(int *p) { *p = *p + 1; }
int main() { f(&x); printf("%d\n", x); }  // prints 8
```

Calling sequence:
```asm
lea r0, x    ; push &x
push r0
bl f
```

Callee dereference:
```asm
ldr r0, fp, 2    ; r0 = p (the address)
ldr r0, r0, 0    ; r0 = *p
add r0, r0, 1
ldr r1, fp, 2
str r0, r1, 0    ; *p = r0
```

---

## ex0803.a — Pass by value-result *(not supported by C)*

The argument is passed by value (a copy is pushed), but on return the callee's final copy is popped back into the caller's variable.

This is *not* a C mechanism — it is shown for comparative purposes.

Calling sequence:
```asm
ld r0, x
push r0
bl f
pop r0       ; retrieve callee's final value of a
st r0, x     ; write it back to x
```

Result: `x` is updated to `a + 1` even though `a` was passed by value — but only if `x` is the sole argument. Aliasing issues can make value-result differ from pass-by-address.

---

## ex0804.a — Pass by name / thunks *(not supported by C)*

Each argument is replaced by a *thunk* — a small function that evaluates the argument in the *caller's* environment and returns its address. The callee calls the thunk to get the current value or address of the argument on each access.

This implements Jensen's device from Algol 60. It is extremely rare and not supported by C or C++.

Assembly pattern:
- A thunk is a short code fragment defined inline in the caller's body.
- The caller passes the thunk's address (via `lea @L1`) as the argument.
- The callee restores the caller's `fp`, calls the thunk with `blr r0`, restores its own `fp`, then dereferences the returned address.

The thunk for `a+b` (where both `a` and `b` may change):
```asm
@L1:  ld r0, a
      ldr r1, fp, -1   ; b (local)
      add r0, r0, r1
      str r0, fp, -2   ; temp storage
      add r0, fp, -2   ; return address of temp
      ret
```

## ex0806.a — Variable-length parameter list (`va_list`)

Implements the C `<stdarg.h>` `va_start` / `va_arg` pattern manually in assembly. The variadic function receives a count as its first argument; a pointer (`va_list p`) walks forward through the stack to pick up successive arguments.

C equivalent:
```c
int add(int argnum, ...) {
    int sum = 0;
    va_list p;
    va_start(p, argnum);
    while (argnum) { sum += va_arg(p, int); argnum--; }
    return sum;
}
int main() { printf("%d\n", add(2, 5, 7)); }  // prints 12
```

`va_start(p, argnum)` assembly: initialize `p` to `fp+3` (one past `argnum`'s slot at `fp+2`).  
`va_arg(p, int)` assembly: dereference `p`, then increment `p` by 1.
