# Chapter 15 — C++ Classes and Objects

Source files from the cuh63 package. This chapter shows how the compiler implements C++ classes: member functions receive a hidden `this` pointer as their first argument, and name mangling distinguishes overloads.

> **ISA reference:** [docs/lcc-isa.md](../lcc-isa.md)

---

## ex1501.a — Structs with free functions (C-style OOP)

A C++ `struct` with free functions that take an explicit `A*` pointer. This is the C-style precursor to true member functions — no `this` pointer magic yet.

C++ equivalent:
```cpp
struct Point { int x; int y; };
void set(A *r, int n, int m)     { r->x = n; r->y = m; }
void display(A *r)               { cout << r->x << r->y; }
int main() {
    A a, b;
    set(&a, 5, 6);  display(&a);
    set(&b, 10, 11); display(&b);
}
```

Both `a` and `b` are stack-allocated (`sub sp, sp, 2` each); their addresses are passed explicitly.

Name mangling: `@set$p1Aii` — `set` taking pointer-to-A, int, int.

## ex1502.a — Objects with member functions (`this` pointer)

A C++ `class` with member functions. The compiler passes the object's address as a hidden first argument (the `this` pointer, at `fp+2`). Member access uses `ldr r1, fp, 2` to get `this`, then `str r0, r1, 0` / `str r0, r1, 1` for the fields.

C++ equivalent:
```cpp
class A {
    public: void set(int n, int m); void display();
    private: int x; int y;
};
A::set(int n, int m)  { x = n; y = m; }
A::display()          { cout << x << y; }
int main() {
    A a, b;
    a.set(5, 6);   a.display();
    b.set(10, 11); b.display();
}
```

Calling sequence for `a.set(5, 6)`:
```asm
mov r0, 6          ; push m
push r0
mov r0, 5          ; push n
push r0
add r0, fp, -2     ; push &a  (this)
push r0
bl @A@set$ii       ; mangled: A::set(int, int)
add sp, sp, 3
```

Name mangling: `@A@set$ii` — `A::set` with two int parameters; `@A@display$v` — `A::display` with void.

## ex1503.a — Dynamic struct allocation with free functions

Same free-function pattern as ex1501, but objects are heap-allocated with an inline `malloc` bump allocator instead of stack-allocated.

```asm
mov r1, 2          ; sizeof(A) = 2 words
bl malloc
str r0, fp, -1     ; p = returned address
```

After `malloc`, the pointer is used exactly as in ex1501: `ldr r0, fp, -1; push r0` before each function call.

## ex1504.a — Dynamic objects with member functions (`new`)

Combines ex1502 (member functions with `this`) and ex1503 (dynamic allocation). Simulates `p = new A` using the inline malloc, then initializes and calls member functions via the `this` pointer convention.

```asm
mov r1, 2         ; sizeof(A)
bl malloc
str r0, fp, -1    ; p = new A

mov r0, 6         ; p->set(5, 6)
push r0
mov r0, 5
push r0
ldr r0, fp, -1    ; push p (this)
push r0
bl @A@set$ii
add sp, sp, 3
```

Mirrors the pattern for stack objects in ex1502, but the base address comes from `ldr r0, fp, -1` (heap pointer) instead of `add r0, fp, -2` (stack offset).
