## ex0301.a — Add two words and display

Loads two `.word` values into registers, adds them, and prints the result with `dout`.

Key instructions: `ld`, `add`, `dout`, `nl`, `halt`, `.word`

```
x: .word 2 / y: .word 3 → prints 5
```

## ex0302.a — String prompt and input

Prompts the user for a string, reads it into a buffer with `sin`, then echoes it back with `sout`.

Key instructions: `lea`, `sout`, `sin`  
Key directives: `.string`, `.zero`

## ex0303.a — Counting loop (sum of first 10 odd numbers)

Uses `mov` to initialize registers and a branch-back loop to sum the first 10 odd numbers (1+3+5+…+19 = 100).

Key instructions: `mov`, `add`, `sub`, `brp`, `lea`, `sout`, `dout`, `nl`

Demonstrates the loop pattern: initialize → body → decrement count → `brp loop`.

---

## ex0304.a — Subroutine call (sub defined before main)

Calls `sub` twice with `bl`; `sub` uses `lea`/`sout`/`ret`.  
Works correctly because `sub` is defined before `main`, so execution begins at `sub` — but `sub` immediately does `ret` with an uninitialized lr, which would crash. The fix is `.start`.

Contrast with ex0305.

Key instructions: `bl`, `ret`, `lea`, `sout`

---

## ex0305.a — Infinite loop (sub defined after main, no `.start`)

Identical logic to ex0304 except `sub` is defined *after* `main`.  
Without `.start`, the CPU begins at address 0 (where `sub` is), which immediately `ret`s into garbage — resulting in an infinite loop or crash.

Demonstrates why `.start` or careful ordering is required.

---

## ex0306.a — Fixing the infinite loop with `.start main`

Adds `.start main` to ex0305, so execution correctly begins at `main` regardless of layout order.

Key directive: `.start <label>`

---

## ex0307.a — `.word <label>` stores an address

Shows that `.word x` assembles to the *address* of `x` (not the value at x).  
`y: .word x` → y holds the address of x; loading y loads that address.

Contrasts with ex0308.

---

## ex0308.a — `.word <number>` stores a literal value

Same program structure as ex0307 but `y: .word 2` stores the literal value 2.  
Loading y loads 2 (not the address of x).

Together ex0307+ex0308 illustrate label vs. literal operands in `.word`.

## ex0309.a — Compare two numbers and branch on sign

Reads two signed integers with `din`, subtracts them, then branches on the result using `brlt`, `bre`, `brgt`, `br`.

Key instructions: `din`, `sub`, `cmp`, `brlt`, `bre`, `brgt`, `br`, `lea`, `sout`

Shows the full three-way signed comparison pattern.

## ex0311.a — Label arithmetic

Demonstrates that labels can be used in expressions: `x+1`, `x+2`, `y-2` compute addresses relative to a label at assembly time.

`z: .word x+2` — z holds the compile-time address of the word 2 past x (which equals the address of y).

Key concept: label arithmetic is resolved by the assembler, not at runtime.

## p0315.a — Linked list with variable-size records (page 315)

Constructs a singly-linked list in static memory. Each record begins with a data word and a `.word` pointer to the next record, with `.zero` padding between records of different sizes.

Illustrates:
- `.word <label>` as a pointer
- `.zero <n>` for struct padding
- Variable-sized records in a linked list
- Null pointer as `.word 0`
