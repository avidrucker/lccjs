## ex0601.a — `while` loop in a non-recursive function

Translates a C `while` loop into the LCC branch pattern: test at the top, `brz` to the exit label, branch back unconditionally at the bottom.

C equivalent:
```c
void nrf(int x) {
    while (x != 0) { printf("%d\n", x); x = x - 1; }
}
```

Assembly loop structure:
```asm
@L0:  ldr r0, fp, 2     ; load x
      cmp r0, 0
      brz @L1            ; exit if x == 0
      ...                ; loop body
      br @L0             ; go back
@L1:
```

Key instructions: `cmp`, `brz`, `br`, `ldr`, `str`

## ex0602.a — Tail recursion

Translates a tail-recursive C function. The recursive call is the last operation before `return`, so no state needs to be saved across the call (though the LCC ABI still pushes/pops a full frame).

C equivalent:
```c
void rf1(int x) {
    if (x != 0) { printf("%d\n", x); rf1(x - 1); }
}
```

Also demonstrates the `s` debugging trap, which displays the current stack — useful for watching the call stack grow and shrink during recursion.

Key instructions: `s` (stack display debug trap), `bl`, `ret`

## ex0603.a — Non-tail recursion

Translates a non-tail-recursive function where work happens *both before and after* the recursive call. The pre-call and post-call output differ, producing a "down / bottom / up" trace.

C equivalent:
```c
void rf2(int x) {
    if (x == 0)  { printf("bottom\n"); }
    else         { printf("down\n"); rf2(x-1); printf("up\n"); }
}
```

Output for `rf2(2)`:
```
down
down
bottom
up
up
```

Key insight: the post-call `sout` ("up") only runs after the recursive call returns, so the stack must preserve the return address across the call — this is exactly what the `push lr / pop lr` frame setup does.
